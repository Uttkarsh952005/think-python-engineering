# Notes on Functions

- **Mental Load**: Functions drastically reduce cognitive load. I don't need to know *how* `print_twice` works when I'm reading `main()`; I only need to know *what* it does.
- **Naming**: Function names should typically be verbs or action phrases (e.g., `calculate_total`, `fetch_user`).
- **Tracebacks**: When a program crashes inside a function, the traceback shows the call stack. Learning to read the stack bottom-up is an essential debugging skill.
