# Chapter 3: Functions

Functions are the building blocks of any complex program. They allow us to name a sequence of statements, making code readable, reusable, and testable.

## Core Concepts
- **Definition vs. Execution**: Defining a function (`def`) does not run it. It only creates the object. Execution requires calling it.
- **Parameters and Arguments**: Parameters are variables inside the function definition. Arguments are the actual values passed during the call.
- **Flow of Execution**: The interpreter jumps to the function, executes it, and returns to where it left off.
- **Local Variables**: Variables created inside a function are destroyed when the function completes.

## Engineering Takeaways
- **Single Responsibility**: A function should do one thing well.
- **Composition**: Complex behaviors emerge from combining simple functions.
